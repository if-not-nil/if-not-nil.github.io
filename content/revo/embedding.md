# embedding revo into your zig project
[docs](docs/basics) | [codeberg](https://codeberg.org/lung/revo) | [github (mirror)](https://github.com/if-not-nil/revo) | [license](#license)

> revo can act like a turing-complete config language, a game engine frontend or any sort of garbage-collected user-facing part of your program

it can be embedded into your project via the `Runtime` struct.
the struct lets you pick your allocator, io, the three stdin/out/err files, and the wrappers make it quite simple to "just run a file"

  </div>

</div>

## basic usage

```zig
const std = @import("std");
const revo = @import("revo");

pub fn main(init: std.process.Init) !void {
    // create a runtime (which owns its own vm)
    var rt = try revo.Runtime.init(init.gpa, init.io);
    defer rt.deinit();

    // compile and execute in one call
    const result = try rt.eval("<test>", "1 + 2 * 3");
    // result is EvalResult (union of .ok | .err)
}
```

## separate compile and execute

very useful for taking advantage of comp

```zig
var rt = try revo.Runtime.init(init.gpa, init.io);
defer rt.deinit();

// compile to artifact
const artifact = try rt.compile("script.rv", source_code);
defer init.gpa.free(artifact.instructions);
defer init.gpa.free(artifact.spans);

// execute artifact
const result = try rt.run("script.rv", artifact);
```

## load from file

```zig
var rt = try revo.Runtime.init(init.gpa, init.io);
defer rt.deinit();

const source = try std.Io.Dir.cwd().readFileAlloc(
    init.io, 
    "script.rv", 
    arena, 
    .unlimited
);

const result = try rt.eval("script.rv", source);
```

# advanced

## persistent globals across evaluations

state is shared across multiple `eval()` calls on the same runtime:

```zig
var rt = try revo.Runtime.init(gpa, io);
defer rt.deinit();

_ = try rt.eval("init", "counter = 0");
_ = try rt.eval("increment", "counter = counter + 1");
_ = try rt.eval("check", "counter = counter + 1");
_ = try rt.eval("print", "print(counter)"); // prints 2
```

## bytecode serialization

compiled artifacts can be serialized to disk and reloaded without recompiling:

lets you take advantage of the comp feature forever

```zig
// first time: compile and cache
var artifact = try rt.compile("config.rv", source);
const bytecode = try revo.bytecode.serialize(rt.vm, artifact, gpa);
try std.Io.Dir.cwd().writeFile(io, "config.rvo", bytecode);

// later: deserialize and run immediately
const cached_bytes = try std.Io.Dir.cwd().readFileAlloc(io, "config.rvo", arena, .unlimited);
const cached_artifact = try revo.bytecode.deserialize(rt.vm, cached_bytes, gpa);
const result = try rt.run("config.rv", cached_artifact);
```
